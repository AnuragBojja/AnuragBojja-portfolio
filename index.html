🏗️ 10-Layer Terraform Architecture
Infrastructure as Code

RoboShop — Full Production Terraform Architecture

10 independently state-managed Terraform layers with S3 remote backends on AWS. Cross-layer resources referenced via SSM Parameter Store — zero hardcoded values. Self-bootstrapping Bastion auto-applies DB and service layers on first boot and triggers Ansible playbook execution per service, achieving fully hands-free provisioning.

  • 6 app services deployed via a single for_each loop driving AMI baking, Launch Templates, ASGs with rolling refresh, and ALB rules.
  • CPU-based target tracking scaling policies for automatic horizontal scaling.
  • Wildcard ACM cert, TLS 1.3 ALB, and CloudFront distribution with optimized caching for static assets and pass-through for API traffic.
Terraform AWS VPC/EC2/ALB/ASG CloudFront SSM Ansible Route53 ACM

⚙️ Ansible Roles Architecture
Automation

RoboShop — Ansible Roles Architecture

Refactored flat per-service playbooks into a full Roles architecture — any service deploys with a single command using $service_name parameterization. Zero code duplication across 10 roles. AWS SSM Parameter Store used for secrets (SSH passwords, MySQL root credentials) via the amazon.aws lookup plugin.

  • All systemd unit files and Nginx configs rendered as Jinja2 templates — no static config files.
  • Single-command service deployment: one parameter, one command, any service.
